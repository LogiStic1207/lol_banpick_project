# LoL Ban/Pick 프로젝트 구조

이 리포지터리는 리그 오브 레전드 팀 밴픽 전략을 자동화하기 위한 기본 틀을 제공합니다.  
주요 목표는 Riot 공식 API를 호출하여 각 소환사의 최근 게임 데이터와 챔피언 사용 빈도를 수집하고, 이를 바탕으로 밴픽 우선순위를 분석하는 것입니다.

## 구조 개요

```
lol_banpick_project/
├── src/
│   ├── riot_api_skeleton.py    # Riot API 호출 및 데이터 수집을 위한 파이썬 스크립트
│   └── app.py                 # Streamlit 기반 UI (밴픽 추천 인터페이스)
├── data/                       # 수집한 원본 데이터와 분석 결과를 저장하는 디렉터리
├── docs/
│   └── project_structure.md    # 프로젝트 구조와 설정 방법을 설명하는 문서
├── requirements.txt            # 필요한 파이썬 패키지 목록
├── .env.example                # 환경 변수 샘플 파일 (RIOT_API_KEY 등)
└── README.md                   # 이 파일 (프로젝트 소개와 구조 설명)
```

### src/
`src/` 디렉터리에는 데이터 수집과 분석에 필요한 파이썬 코드가 위치합니다.  
`riot_api_skeleton.py`는 Riot API를 통해 소환사의 PUUID를 조회하고, 지정한 큐 타입의 매치 기록을 가져와 챔피언 사용 통계를 계산하는 기능을 제공합니다.

또한 `app.py`는 `streamlit`으로 작성된 웹 인터페이스로, 사용자가 소환사 목록과 큐 타입을 입력하고, `data/` 디렉터리의 스크림 데이터를 선택하거나 업로드하여 분석을 수행할 수 있게 합니다. UI에서는 가중치 설정과 결과 표출을 위한 기본 틀을 제공하며, 추후 밴픽 추천 로직을 이곳에 통합할 수 있습니다.

### data/
데이터 수집 결과와 분석 결과를 저장하는 공간입니다. 예를 들어 CSV 파일로 저장하거나, pickle/JSON 형식으로 저장할 수 있습니다.  기본적으로 `scrim_data.xlsx`라는 예시 스크림 데이터 파일이 포함되어 있으며, 이는 연습 경기의 챔피언 픽·밴과 승패 정보를 담고 있습니다.  UI에서 이 파일을 읽어 챔피언 사용 빈도와 승률을 요약할 수 있습니다. 사용자는 최신 스크림 데이터를 이 디렉터리에 추가하여 분석에 반영할 수 있습니다.

### docs/
프로젝트와 관련된 설명 문서 및 기술적 참고 자료를 보관합니다. 여기에는 프로젝트 구조, API 사용 방법, 개발 시 유의사항 등이 포함됩니다.

### requirements.txt
프로젝트 실행에 필요한 파이썬 의존성을 정의한 파일입니다. `pip install -r requirements.txt` 명령으로 필요한 패키지를 설치할 수 있습니다.

### .env.example
API 키와 같은 민감한 정보를 저장하는 `.env` 파일의 예시입니다. 실제 사용 시에는 `.env.example`을 `.env`로 복사하고, `RIOT_API_KEY` 값을 본인의 키로 설정하세요.

## 시작하기
1. **레포지터리 준비**: 이 디렉터리 구조를 그대로 로컬 또는 클라우드 환경에 복제합니다.  
2. **환경 설정**:
   - `python -m venv venv`와 같은 명령으로 가상환경을 만들고 활성화합니다.  
   - `pip install -r requirements.txt`로 의존성을 설치합니다.  
3. **API 키 설정**: `.env.example`을 복사하여 `.env` 파일을 만들고, `RIOT_API_KEY=your_key_here`를 채웁니다.  
4. **데이터 수집**: `python src/riot_api_skeleton.py`를 실행해 예시 소환사의 챔피언 사용 통계를 수집해 봅니다. 필요에 따라 `PLAYERS` 리스트와 큐 타입을 수정하십시오.  
5. **확장**: 자신의 프로젝트에 맞게 모듈화하고, 결과를 분석하는 코드나 밴픽 로직을 추가합니다.
6. **UI 실행**: 웹 기반 인터페이스를 사용하려면 `streamlit run src/app.py` 명령을 실행합니다. 브라우저가 열리면 소환사 목록 입력, 큐 타입 선택, 스크림 데이터 업로드/선택 등을 통해 API 데이터와 스크림 데이터를 함께 분석할 수 있습니다. 분석 결과는 표 형태로 표시되며, 향후 밴픽 추천 알고리즘을 추가하여 사용할 수 있습니다.

보다 자세한 설명은 `docs/project_structure.md`에서 확인할 수 있습니다.

## VS Code 통합

이 프로젝트는 Visual Studio Code 사용자를 위한 편의 설정을 제공하기 위해 `.vscode` 폴더를 포함하고 있습니다.  이 폴더에는 다음과 같은 설정이 들어 있습니다:

- **launch.json** – `src/riot_api_skeleton.py` 스크립트를 간편하게 디버그할 수 있는 구성이 정의되어 있습니다.  스크립트를 실행하면 `.env` 파일에 저장된 Riot API 키가 자동으로 환경변수로 설정됩니다.
- **settings.json** – 프로젝트별 Python 인터프리터와 `.env` 파일을 지정하고, 저장 시 자동 코드 서식을 활성화합니다.  또한 VS Code의 통합 터미널이 `.env` 파일을 읽도록 합니다.
- **tasks.json** – 반복적인 명령을 작업(Task)으로 정의합니다.  “Run Riot API script” 작업은 스크립트를 터미널에서 실행하고, “Install dependencies” 작업은 `requirements.txt`에 명시된 의존성을 설치합니다.

VS Code 환경 설정 및 사용법에 대한 자세한 단계는 `docs/vscode_setup.md`에서 확인할 수 있습니다.
