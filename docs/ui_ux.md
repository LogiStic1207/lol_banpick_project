# UI/UX 가이드

이 문서는 `lol_banpick_project`에서 제공하는 Streamlit 기반 웹 인터페이스(`src/app.py`)의 사용 방법과 확장 방향을 안내합니다.  UI는 데이터 분석과 밴픽 추천 로직을 시각적으로 실행하기 위한 발판을 제공합니다.

## 기능 요약

1. **소환사 목록 입력**: 좌측 사이드바에서 분석할 소환사 이름과 태그를 입력할 수 있습니다. 최대 10명의 소환사를 한 번에 분석할 수 있습니다.
2. **큐 타입 선택**: 랭크 솔로/듀오(420), 랭크 플렉스(440), 일반(430) 세 가지 큐에 대한 데이터를 포함할지 체크박스로 지정할 수 있습니다. 분석에 사용할 최대 경기 수를 슬라이더로 설정합니다.
3. **스크림 데이터 선택 또는 업로드**: `data/` 폴더 내에 존재하는 엑셀 파일(`.xlsx`, `.xls`)을 드롭다운에서 선택하거나, 사용자가 새로 업로드할 수 있습니다. 엑셀 파일은 `스크림 데이터`라는 시트에 데이터가 있어야 하며, 표준 컬럼(경기, 진영, 탑, 정글, …, 블루 벤, 레드 벤)을 가진 구조여야 합니다.
4. **분석 실행**: “분석 실행” 버튼을 누르면 Riot API를 통해 각 소환사의 최근 경기 데이터를 가져오고, 선택된 큐에 대한 챔피언 사용 통계를 계산합니다. 또한 선택/업로드된 스크림 데이터를 읽어 팀과 상대 챔피언의 사용 빈도 및 승률을 요약합니다.
5. **결과 표시**: 각 소환사별로 API 데이터에서 계산된 챔피언 통계를 표로 보여 주며, 스크림 데이터에서 추출한 우리 팀과 상대 팀의 사용 빈도 및 승률도 별도의 표로 표시합니다.

## 확장 방향

- **밴·픽 추천 알고리즘 통합**: 현재 UI는 데이터 요약까지만 제공합니다. 향후 알고리즘을 구현하여 API 데이터와 스크림 데이터를 결합하고, 가중치를 적용해 밴픽 우선순위를 계산하는 함수를 만들 수 있습니다. 이 함수를 `app.py`에 호출하여 결과를 사용자에게 보여 줄 수 있습니다.
- **시각화 향상**: Streamlit의 차트 기능을 활용하여 승률, 사용 빈도 등을 그래프로 표현하면 더 직관적으로 데이터를 파악할 수 있습니다.
- **포지션별 분석**: 특정 포지션(탑, 정글 등)별로 필터링하거나, 팀 전체 조합의 성능을 분석하는 UI 요소를 추가할 수 있습니다.

## 실행 요령

1. 가상환경을 활성화하고 프로젝트 루트에서 `pip install -r requirements.txt`를 통해 의존성을 설치합니다.
2. `.env` 파일에 `RIOT_API_KEY`를 설정합니다.
3. 다음 명령으로 Streamlit 앱을 실행합니다:
   ```bash
   streamlit run src/app.py
   ```
4. 브라우저에서 열리는 페이지에서 좌측 패널을 통해 소환사와 스크림 데이터를 설정하고 분석을 시작합니다.

이 UI는 빠르게 프로토타입을 만들기 위한 것이며, 향후 프론트엔드 프레임워크(React 등)와 백엔드 API를 이용해 확장형 서비스를 구축하는 것도 가능합니다.
