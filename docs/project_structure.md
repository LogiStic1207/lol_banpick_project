# 프로젝트 구조 설명

이 문서는 `lol_banpick_project` 폴더 내 파일과 디렉터리의 역할을 상세히 설명합니다.  
이 구조는 리그 오브 레전드 밴/픽 추천 프로그램 개발을 위한 기본 골격이며, 필요에 따라 추가 모듈을 확장할 수 있습니다.

## 디렉터리 및 파일

- **src/**  
  애플리케이션의 핵심 코드를 담는 폴더입니다. 현재 `riot_api_skeleton.py`가 포함되어 있으며, Riot API를 호출하는 기본 함수와 예제 `main()` 함수가 구현되어 있습니다. 향후 밴픽 알고리즘, 데이터 파싱/정렬 로직 등도 이곳에 추가할 수 있습니다.
  
  추가로 `app.py` 파일이 있습니다.  이는 `streamlit`을 기반으로 한 간단한 웹 인터페이스로, 소환사 목록을 입력하고 API 데이터를 조회하는 동시에 `data/` 폴더의 스크림 데이터를 불러와 요약 통계를 보여 주는 역할을 합니다.  이 UI는 가중치 계산과 밴픽 추천 로직을 통합할 발판으로 사용할 수 있습니다.

- **data/**  
  수집한 매치 데이터와 가공된 결과를 저장하는 공간입니다. 예를 들어, 각 팀의 챔피언 사용 빈도를 CSV로 저장해 분석에 활용할 수 있습니다.

  예시로 `scrim_data.xlsx` 파일이 포함되어 있습니다.  이는 최근 스크림(연습경기) 기록을 엑셀 형식으로 저장한 것으로, `app.py`가 이를 읽어 챔피언 사용 빈도와 승률을 요약합니다.  사용자는 자신의 최신 스크림 데이터를 이 폴더에 추가하여 분석에 반영할 수 있습니다.

- **docs/**  
  문서화 관련 파일을 저장합니다. 프로젝트 구조, 설정 방법, 개발 가이드 등을 포함합니다.

- **requirements.txt**  
  프로젝트 실행에 필요한 파이썬 패키지를 나열한 파일입니다. 예시로 `requests`, `pandas`, `python-dotenv`, `streamlit` 등이 포함되어 있습니다.  `streamlit`은 UI를 구동하는 데 사용됩니다.

- **.env.example**  
  `.env` 파일의 예시 템플릿입니다. 실제 운영 시에는 `.env.example`을 `.env`로 복사하고, `RIOT_API_KEY`에 본인이 발급받은 키를 입력합니다. 예시:
  ```
  RIOT_API_KEY=YOUR_RIOT_API_KEY_HERE
  ```

- **README.md**  
  프로젝트의 개요 및 전체 구조를 간단히 설명하는 파일입니다.

- **.vscode/**  
  VS Code 사용자 편의를 위해 포함된 디렉터리입니다.  `launch.json`에는 디버깅 설정이 정의되어 있으며, `settings.json`은 프로젝트에 맞는 파이썬 인터프리터와 `.env` 파일을 설정합니다.  `tasks.json`은 스크립트 실행과 패키지 설치를 반복 작업으로 정의해 단축키 한 번으로 실행할 수 있게 해 줍니다.  자세한 VS Code 설정과 사용법은 `docs/vscode_setup.md`를 참고하세요.

## 초기 단계 제안

1. **환경 준비**  
   파이썬 3.x 환경을 준비합니다. virtualenv 또는 conda로 격리된 환경을 만들고 `requirements.txt`에 정의된 패키지를 설치하세요.
2. **API 키 설정**  
   Riot 개발자 포털에서 API 키를 발급받습니다. `.env` 파일을 만들고 `RIOT_API_KEY` 값을 설정합니다.
3. **테스트 실행**  
   `src/riot_api_skeleton.py`를 실행하면 예제 플레이어 리스트에 대한 데이터 수집이 진행됩니다. 여기서 원하는 소환사 명단과 큐 타입을 지정하면 됩니다.
4. **데이터 저장 및 분석**  
   `summarise_champion_usage()` 함수에서 반환된 `DataFrame`을 CSV로 저장하거나, 추가 분석을 위한 로직을 추가합니다.
5. **밴픽 로직 통합**  
   수집된 챔피언 빈도 데이터를 기반으로 밴/픽 우선순위를 계산하는 알고리즘을 설계하고 구현합니다. 이는 팀의 모스트 픽, 상대팀 픽, 메타 데이터를 고려해 가중치를 부여하는 부분으로 확장될 수 있습니다.

향후에는 라이브 경기 데이터를 실시간으로 반영하거나, 머신러닝 모델을 적용해 밴픽 전략을 고도화하는 것도 가능합니다. 하지만 우선은 API 호출과 데이터 수집/정리에 익숙해지는 것이 중요합니다.
